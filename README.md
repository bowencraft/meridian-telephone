# Telephone

一款发生在伦敦雨夜公共电话亭中的 2D 拟物叙事解谜游戏。

玩家通过提起听筒、真实拖动转盘、接听主动来电、检查电话亭线索并选择通话回应，逐步追查 **The Meridian Courtesy Exchange（子午礼仪交换所）** 如何利用公共电话网络改写接听者的语言与身份。

Telephone 不是 Typewriter 的换皮，而是“单物品强交互 + 事件驱动节点图叙事 + 可视化内容后台”工具链的第二个完整游戏样本。

## 页面

- `#/`：Telephone 前台游戏
- `#/admin`：可视化剧情交换台
- `#/record`：通话、拨号、线索与多周目结局档案

## 主要交互

- 点击听筒即可拿起，之后听筒会悬浮跟随指针；线长限制、角度摆动和动态卷线会同步响应。
- 把听筒移回挂架会出现磁性吸附反馈；再次点击、点击非交互背景或按回车可放回听筒。
- 将数字孔顺时针拖至挡片，等待转盘回弹后才会确认数字。
- 数字键可作为键盘无障碍拨号路径；`0` 的回转距离最长。
- 七位号码或紧急号码 `999` 会自动尝试接通。
- 电话会在待机时主动响铃；未及时接听会记为未接来电。
- 通话中的云朵选项支持显式条件、flags、跨周目结局条件和超时分支。
- 墙上的卡片、广告、铭牌、报纸、电话簿与退币槽可以发现号码或隐藏线索。
- 号码簿只展示当前周目或历史周目已经发现的线路。
- 所有声音由 Web Audio API 实时生成，可随时静音。

## 内容规模

- 40 个剧情、系统、场景与结局节点
- 74 条事件驱动转场
- 8 个有效号码
- 3 通主动来电
- 6 个电话亭热点
- 7 个结局（含跨周目隐藏路线）
- 多组通话 variants、顺从 / 怀疑 / 错号 / 挂断等状态记录

主要号码包括天气服务、Meridian 咨询与投诉、注销的内部席位、中央交换机、失物招领、午夜广播与紧急服务。核心成功路线需要从投诉线路取得权限，找到被注销的十九号席位，再进入中央交换机切断训练线。

## 剧情后台

`#/admin` 基于 `@xyflow/react`，支持：

- 创建、移动、连接和删除电话剧情节点
- 编辑节点类型、通话 variants、fallback、LCD 文本、挂断权限与结局类型
- 编辑拨号、选项、来电、挂断、超时、场景检查和自动事件
- 编辑条件、effects、flags 与优先级
- 管理有效号码、电话超时与场景热点
- 模拟号码、来电、选项、超时和场景事件，查看命中边、候选边、flags 与当前节点
- 导入 / 导出完整 JSON
- 本地开发写回源码；静态环境回退到 `localStorage`

唯一剧情源文件为 [`src/story/telephone.rules.json`](src/story/telephone.rules.json)。

## 持久化

游戏只在设备本地保存：

- 最后一次完整通话
- 最近 16 次结局记录
- 已发现号码与线索
- 已归档结局和尝试次数
- 生产环境中的剧情后台覆盖版本

档案页可一键清除所有本地进度并重新开始。
