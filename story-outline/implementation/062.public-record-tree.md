# Seedline新闻、广播与公开记录树

## Implementation Model

当前没有独立news schema。公开记录以“条件场景物件 + 可选电话线路”实现：报纸、广播节目单、工会简报、版本卡先显示粗略/样式/细节；带号码者解锁一个离散电话树，不带号码者只写入durable fact或clue。

新闻不会承载唯一主线钥匙。第四章的Radio Nocturne号码由Leonard固定口述，场景里的节目单只是第二来源。

## Side Telephone Seeds

### Deptford旧诊所 — 3372181

第一次拨打是停用线路上的值班录音：当年锅炉援助如何分流，结尾说明诊所从未把病人名单上传中央。第二次拨打由退休接待员短暂接听，她不认识Vale，只记得“工具允许我们自己改一句话”。用于证明独立人物并非主角人格。

### Ashdown公开调查 — 4051979

自动档案提供两段互相补充的摘要：官方主报告强调部门词汇不兼容；工会附件指出部门知道伤者位置但争执谁承担车辆费用。它解释Wren为何真诚相信统一格式，又为何他的版本不完整。

### 地方分支互助台 — 6621904

三个地方维护者轮值：一人欢迎中央兼容表，一人拒绝礼仪层，一人只关心旧交换机是否还有零件。玩家不能说服全体；该线为`operator`结局展示持续维护成本。

## Object Copy Contract

每件无号码公开记录都填写：

- `copy.summary`：它是什么、来自哪个现实机构或日期。
- `copy.style`：纸张、油墨、折痕、盖章、手写修订等可观察样式，不推断秘密。
- `copy.firstVariants`：细读后能获得的具体事实及其限制。
- `copy.repeatVariants`：重看时提醒它改变的是哪项判断，不重复百科解释。

首次检查卡同时显示summary、style与按seed选定的firstVariants细节；复查仍保留同一summary和style，只把“细节”一栏替换为repeatVariants。这样三层信息始终可比较，不把样式说明误当作一次性剧情文本。

带号码物件另外填写printedLines和phoneRefs。表面号码、号码簿条目和拨号节点必须引用同一phone ID。

## Safety Rules

- 新闻只报告已发生、可公开记录的后果，不替人物读取内心。
- 不用报纸一次解释Seedline、分区、礼仪层和双签名。
- 每则记录只增加一个世界判断；标题可懂，细节可选。
- 结局后新闻承认真实收益与真实代价，避免“好结局报喜、坏结局嘲讽”。
- 场景检查的正文必须写在prop copy；`scene` node只承载可验证effects，不承载玩家唯一可见文本。
