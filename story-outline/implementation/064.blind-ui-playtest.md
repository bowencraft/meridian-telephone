# Seedline纯UI盲测记录与最终验收门

## Test Contract

盲测者不得阅读源码、JSON、测试、`story-outline/`或剧情说明，只能从一个空浏览器档案打开玩家页面。首轮必须从玩家可见的初始来电和物件开始；完成结局后重开，再取得至少两个不同结局，并核对人物、事实、号码与物件后果是否延续。

## Invalidated Run — Legacy Override Discovery

第一轮盲测发现并修复了真实交互问题，但叙事结果作废：浏览器中的`telephone.storyOverride.v3`无条件覆盖仓库默认JSON，旧`telephone.progress.v1`又把旧周目带进运行时。盲测者因此玩到“天气校准、语音训练、失物身份”，而不是六章Seedline。该轮抵达断线、永远转接与入职，只能证明旧图可玩，不能证明本目标完成。

这次失败说明：空的玩家进度不等于空的剧情覆盖。以后盲测必须同时核对首个来电名称与首章人物，不能仅看首页标题。

## Findings Retained From The Invalidated Run

- 首版转盘只支持难以发现的拖动，数字键还依赖焦点；已增加逐孔单击与全局数字键。
- 快速连点或正常速度键入会静默吞掉回弹期间的数字；已增加顺序输入队列。
- 来电可能在号码簿或物件说明弹窗后方开始；现仅在无弹窗的待机状态排程。
- 来电窗口改为18秒；接近末声接听仍由ring-cycle防竞态保护。
- 拿取硬币与成功投币说明只首次弹出，后续不再重复打断拨号。

## Runtime Isolation Fix

- 新规则覆盖键：`telephone.storyOverride.seedline.v1`。
- 新进度键：`telephone.seedline.progress.v1`。
- 新最近记录与档案键：`telephone.seedline.lastRecord.v1`、`telephone.seedline.recordArchive.v1`。
- `telephone.storyOverride.v1/v2/v3`不再参与运行时加载；旧进度与旧档案不会被Seedline读取。
- 新命名空间仍允许后台保存与读取新版自定义规则。

## Browser Recheck Before Second Blind Run

- 新空档案首个来电显示“圣西普里安医院急线”，不再出现旧“雨况/天气校准”。
- 物件说明打开超过5秒时没有在背后开始来电；关闭后重新取得完整延迟并响铃。
- 在听筒保持焦点时快速输入`9460264`，队列逐位回弹并正确接通道路状态线，没有形成`8100`或未知线路。

## Final Acceptance Gate

第二轮有效盲测必须同时满足：

1. 首轮从Maeve医院来电开始，并沿号码交接完成六章，而非旧图或跨周目捷径。
2. 能复述每章当前任务、下一号码的来源与失败后恢复方法。
3. 至少抵达三个不同的Seedline结局，并确认重开是后续复核案卷、不是时间循环。
4. 至少一名人物明确承认上一夜的决定或后果，Clara/私人恢复在结局中有去向。
5. 快速拨号、错号、挂断、漏接、物件检查和重开均无软锁或静默吞输入。
6. 若报告FAIL，继续修改并重新盲测；不得用自动图遍历代替玩家UI验收。
