# Seedline跨周目状态与记忆表

## World Explanation

结局后的重新开始是更晚的一夜。Vale私人侧再次按治疗预约来到MCE-0，三音声学检索钥匙恢复同一大脑对工作记忆的访问；交班磁带重放Vale主动留下的摘要。人物通过自己的记忆、工单和现实后果记得上一夜，电话亭不重置世界，也不上传人格。

## State Layers

| Layer | Type | Lifetime | Examples |
| --- | --- | --- | --- |
| UI/mechanical | React state | 当前挂载 | 听筒、硬币拿起、弹窗、拨号缓冲 |
| Runtime flags | scalar map | 当前完整游玩 | 当前章节、四秒窗、已听录音菜单、发布门临时状态 |
| Durable facts | string set | 结局保存后跨周目 | “曾经发生过”的帮助、背叛、损坏与版本后果 |
| Durable state | scalar map | 结局保存后按key覆盖 | 当前签名、来源卡、令牌、医院证据和发布状态 |
| Discoveries | number set | 结局保存后跨周目 | 已登记电话号码 |
| Clues | string set | 结局保存后跨周目 | 档案中显示的文字证据名 |
| Seen endings | ending set | 永久直到清档 | 七种结局 |
| Attempts | integer | 每个完成结局+1 | 新闻、交班和重拨语气演化 |

Durable facts与durableState进入`CallRecordData`、`ProgressData`和`RuntimeState`。`addFact`去重；`hasFact(expected:false)`可以判断尚未发生；`setDurable`按key覆盖，`durableEquals`判断当前状态；`lastEnding`单独保存上一夜结局。不得用可重复increment表达长期关系。

## Save Semantics

只有进入ending node才保存完整长期记录。游戏必须在结局前明确提示“中途离开不会形成交班”；这不是惩罚，而是MCE-0只有封存通话才能写入下一盘磁带。开发测试必须验证：

1. facts与号码、clues、endings一起合并去重；durableState按key覆盖；lastEnding写本局ending。
2. runtime reset清除临时flags但恢复facts、durableState与lastEnding条件。
3. clearProgress同时清空facts。
4. 旧版没有facts的localStorage可以安全迁移为空数组。
5. 固定交班来电与人物重拨能读取上一结局；opening仍是固定电话亭文本，不依赖未实现的auto路由。
6. 同一progress不清档连续归档多个不同结局；每轮处理新的复核docket。

## Privacy Boundary

长期事实只记录可被Vale交班、公共记录或对方本人记住的行为。它不能记录无人观察的心理状态；例如不用`wren-afraid`，只用`wren-legal-challenge-issued`。这让“人物记得”保持现实而不是系统全知。
