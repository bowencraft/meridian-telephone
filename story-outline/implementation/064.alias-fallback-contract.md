# Seedline号码Alias、Fallback与防死路契约

## Number Normalization

所有输入先移除空格、连字符、括号和非数字字符。实体转盘仍只产生数字。canonical号码和每个alias必须为三位紧急号或七位本地号，且全目录不可互相冲突。

`PhoneDirectoryEntry.aliases`只用于旧印刷号或同一线路迁移号，不创建第二个号码簿联系人。引擎把alias规范化为canonical后再匹配`dialNumber`边；拨号记录同时保存玩家实际输入和联系人label，界面显示canonical。

| meridian-team-227 | 8714227 | 8714027 | 公共菜单旧路由与夜班直线 |
| founder-archive | 8714036 | 8714136 | 外部托管卡与Meridian内部迁移号 |
| meridian-release | 8714127 | 8714027不得复用 | 发布台无共享alias，避免与Peter线冲突 |

最后一行明确禁止冲突；若历史文案需要相似号码，用文本解释而不注册alias。

## Explicit Dial Fallbacks

每个directory号码至少有一条`from: global`的dial边。条件化号码使用以下优先级区间：

- `900–999`：结局后或完成后回访。
- `700–899`：当前章节合法入口。
- `400–699`：修复/补证入口。
- `100–399`：提前拨打的说明节点。
- `<100`：通用系统fallback。

低优先级提前拨边不得设置wrongDials；它必须说明：

1. 该号码属于谁。
2. 目前缺哪一份具体材料。
3. 应拨哪个已知或当场读出的恢复号码。
4. 现在挂断不会失去什么。

## Event Fallbacks

| Event | Required authoring |
| --- | --- |
| choice | 条件OR可拆成相同trigger.value的平行边；`getChoices`按value只显示最高优先级胜出边，validator要求同组可见文案一致 |
| timeout choice | 给出人物合理的沉默反应；若进入结局必须明确之前已建立风险 |
| timeout call | 回到可挂断节点或自动挂断，不推进核心证据 |
| hangUp | 大多数通话回idle并退审计币；只有第六章已确认的最终动作可在挂断触发结局 |
| incomingAnswer | 每个RingEvent ID恰有一条可达入口；`nodeId`必须与目标一致 |
| sceneInspect | 有sceneEvent才需要边；正文在prop copy，边只做幂等effect |

不使用正式UI不会dispatch的`auto`与`keywordAny`承载主线。

## Variant Contract

| Field | Meaning | Minimum |
| --- | --- | --- |
| node `variants` | 同一事实与行动问题的不同措辞 | 主线节点2条；结局节点1条可接受 |
| node `fallbackVariants` | 非拨号事件没有边时，重申当前等待 | 所有menu/call节点1–2条 |
| prop `firstVariants` | 首次细读得到的同一事实变体 | 重要物件2条，普通物件1条 |
| prop `repeatVariants` | 重看后的简短提醒或结局响应 | 每件至少1条 |

Variants不得随机改变：日期、号码、人物态度、病人结果、条件是否满足或下一行动。

## Hangup Recovery

玩家随时可挂断，以下情况除外：已经选择最终签署/撤销且节点标注`canHangUp:false`。每次外拨写一次`refundableCall=true`；唯一`closeCall`生命周期在挂断、超时、错号、忙线或结局关闭线路时检查并清零，所以一通电话最多退一次。关闭时：

- 当前电话节点回`booth_idle`。
- 合法取得的runtime facts/numbers保留在本轮。
- MCE-0审计币从退币槽返回。
- 该号码重拨进入最近稳定检查点，不从整章开头重复。
- 主动来电若漏接，以固定号码或稍后的补救来电恢复。

## Wrong And Busy Numbers

未知号码进入`wrong_number`，明确提示核对号码，不暗示机器读心。登记号码在任何条件下都由显式边处理，因此`busy_line`只用于真实占线的支线或维护时段。

错号不再触发“号码磨损”结局。号码磨损来自玩家在第六章明确选择物理破坏安全签名卡；错误拨号只记录操作统计，并退还审计币，避免资源软锁。

## Reachability Assertions

自动测试必须生成并保存见证路径：

1. 空档案到六章发布门。
2. 七结局各一条真实事件序列。
3. operator和counterfeit各自的缺条件与补条件路径。
4. 每个主号码提前拨、完成后重拨、alias拨入。
5. 漏接Maeve后从医院号码恢复。
6. 每个主线节点挂断后可重拨。
7. 无任何概率物件时仍能完成主线。
8. 连续错拨后仍有硬币继续。
9. 所有RingEvent目标存在且有incomingAnswer边。
10. 所有phoneRefs、discoverPhone、aliases、sceneEvents和条件fact都有生产者。
11. 同一ProgressData连续归档多个结局，lastEnding与当前durableState始终指向最近一夜。
12. `sourceCard released → damaged → blocked → rebuilt → open`与`peterToken available → spent`完整状态序列。

任何测试不得直接`setFlag`伪造故事条件，也不得手工向结果集合加入结局名。
