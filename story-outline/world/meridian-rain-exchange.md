# 1989年伦敦与Seedline夜班网络

## Description

### 读者最先需要知道的五件事

1. 电话另一端都是现实中的人、录音或明确标记的旧档案，不存在无所不知的电话意识。
2. Meridian能改写工单和话术，不能让雨停下，也不能凭一句话抹掉陌生人的人格。
3. 故事中的“你”是Evelyn Vale被工作记忆分区隔离出来的夜班自我，与私人自我共用一具身体。
4. 每次重新开始都是下一次夜班。只有当班权限重置，昨日发生的帮助、欺骗、损坏和伤害都继续存在。
5. Seedline已经被许多机构复制。最终问题不是找到一根万能电缆，而是决定谁有权发布下一版规则。

### 时代与城市

故事发生在架空的1989年伦敦。红色公共电话亭仍遍布街角，微型磁带、热敏纸、拨号盘、穿孔卡与绿色字符终端和早期数字交换机并存。公共服务正在把夜间接线、转运协调和投诉处理外包给私人承包商。办公室里每个人仍使用纸质工单，但一个版本号已经可以同时改变医院、交通与呼叫中心的处理顺序。

这不是复古装饰。旧模拟网让独立故障台、社区广播和纸面托管还能保存不受Meridian统一更新的证据；新数字网又让同一套错误路由迅速复制到多家机构。城市正好卡在“地方还能留下痕迹、中央已经可以规模化控制”的时间点上。

### Seedline的原始目标

1979年，公共通信研究组成立。成员包括语言系统研究员Evelyn Vale、紧急协调主管Alistair Wren和线路工程师Leonard Campbell。此前的Ashdown铁路事故中，医院、交通和消防使用互不兼容的表格与措辞，一名伤员被三个部门反复转接。Wren亲手接过其中一通迟到的电话；Vale则负责事后逐字分析那些记录。

团队创造了Seedline：一套可以复制的模块化线路包。医院可以从基础包长出转运分支，交通部门可以增加封路分支，社区热线也能保留自己的人工入口。它的设计原则是“来电者不必为了找到负责人而重复一场危机”。早期Seedline确实缩短过救援时间，也让小型机构不必等待中央采购。

1984年公共部门削减研究预算后，Wren把团队和专利带入新成立的Meridian Courtesy Exchange。地方自由扩展原本是Seedline的优势，却让不同分支开始产生不兼容版本。基础包可以从磁盘或线路副本复制，每个地方分支保存自己的路由树和版本卡。联网分支在夜间拨号同步时下载一张“兼容表”，据此决定哪些医院、交通和投诉分支可以互相交接。Wren认为只有中央认证能防止下一场Ashdown事故；政府和企业也愿意为一个能显示“分歧减少”的承包商付费。

原团队为了防止单人接管，把发布权拆成两部分：Wren持有运营签名，确认合同范围和兼容目标；Vale持有安全签名，确认投诉、拒绝和人工入口没有被删除。两项签名同时出现，联网分支才会在夜间同步中自动接受新兼容表，公共合同也才承认该版本。没有Vale签名，Wren仍能更新Meridian自有楼层，却不能让大多数公共分支自动跟随；没有Wren签名，Vale可以撤销或公开安全说明，却不能假装代表现有合同发布统一版本。离线副本与主动拒绝中央同步的地方分支始终可能保留旧版本。

### 礼仪层与制度恐怖

Meridian在Seedline上增加礼仪层。它不直接控制思想，而是把组织权力写进处理流程：客服若允许来电者沉默会被扣绩效；护士若使用未认证词语，工单会降为“需要确认”；故障台若没有完整编号，事件不会进入人工复核；管理者可以把真实风险改名为“无服务影响”。

对普通公众，礼仪层只利用权威、疲惫和重复。一个人仍然能拒绝，但拒绝会让菜单缩短、等待变长或服务资格消失。它最危险的地方不是魔法，而是每一次决定看起来都像普通合规操作。

### 工作记忆分区

Vale后来提出一种高风险员工保护实验：让创伤热线与夜班接线员只在特定声学提示下访问工作记忆，下班后恢复私人记忆。她最初希望员工不必把每一场危机带回家，也让机构无法在非工作时间调用受训判断。

分区只能用于经过数月校准、主动签署过参与文件的人。它不会制造新人格，也不会复制身体；它只是把同一人的记忆访问切成两个不对称区域。私人自我知道自己在哪里工作，却记不起具体通话；工作自我记得岗位和夜班，却触碰不到家庭与日常生活。

Meridian把这项保护变成管理工具。私人自我无法证明夜班受过何种对待，工作自我也无法因为家庭或法律后果轻易辞职。Peter Ward是现役分区员工；Evelyn Vale是第一位长期测试者，也是最严重的失败案例。

### MCE-0与“重新开始”

MCE-0原本是Seedline的外部审计终端，一座接在旧公共交换机上的红色电话亭。它不受Meridian主版本自动更新，供研究者在中央系统失效时提交人工故障。Vale在这里安装了一只微型交班磁带。磁带不保存或上传人格：工作记忆始终存在于同一具身体的大脑中，只是在提示音关闭后无法访问。磁带记录的是重新打开相同访问范围所需的声学检索钥匙、Vale自己的口述交班、已确认号码和正式线索。

故事开场前，Wren借一次强制“治疗交接”把Vale的工作记忆限制在MCE-0。她的私人自我仍会按照日程卡前往电话亭；听筒接通的提示音和磁带检索钥匙响起后，故事中的“你”重新获得对既有夜班记忆的访问。闭线时提示音关闭，私人自我接管身体并离开。下一次预约，同一大脑重新打开同一组记忆；口述交班帮助Vale确认昨天的决定，没有任何新人格被制造。

因此结局后再开始有清楚边界：伦敦日期前进；梅芙记得病人是否被延误；Peter记得自己是否被出卖；Leonard的工单仍在；Dorothy的证物不会复原；Wren会从质检记录中学习。每天清零的只有当班菜单位置、临时人工许可和未登记草稿。

### 现在发生的危机

Meridian准备向全国公共服务发布礼仪层4.0兼容表。凌晨四点，联网分支会拨号同步；只有同时带有Wren运营签名与Vale安全签名的版本会被自动接受。与此同时，未经中央批准的Seedline分支已经被医院、工会和社区热线复制。部分地方分支安全，部分仍携带会压低真实投诉的旧规则，离线副本也不会听从中央撤销。Wren需要Vale的签名来推动大多数公共分支统一更新；Vale若只撤掉中央签名，危险副本仍会继续运行，却能阻止新的礼仪层覆盖全网。

六章故事发生在版本发布前的六个连续夜班。第一章的一场医院转运把抽象制度变成眼前后果；之后每一章都补上一种做出最终决定所需的现实权力：受害事件、员工证词、技术工单、外部证据、创始人签名和公开治理选择。

## Story Pressure

### 人物为什么不能直接离开

- Evelyn的工作自我只能在MCE-0提示音下行动；她既想与私人自我重新连接，又知道直接接受Wren的整合会把签名交回Meridian。
- Maeve必须完成Thomas Osei的夜间转运。绕开承包线路可能救人，也可能让她承担擅自调度的职业责任。
- Peter的私人自我依赖Meridian工资和母亲的医疗计划，夜班工作自我却越来越清楚礼仪层在伤害别人。
- Leonard若未经可审计程序停机，Meridian会把事件定性为破坏并从备份恢复；他需要一条能经得住调查的证据链。
- Dorothy受托保管创始人材料。声纹可以重放，情绪可以伪装，只有来源、日期和双重凭据能让她合法放行。
- Wren既要保住合同和机构控制，也真诚相信失控分支会重演公共事故；他会牺牲个人自主性来换取全网一致。

### 六夜压力升级

1. 一名病人的转运是否被错误天气记录延误。
2. 一名员工是否为了保存证据冒着失去收入的风险。
3. Vale是否承认自己是分区技术的设计者，而非只有受害者身份。
4. 外部托管是否能在Meridian发布新版前完成核验。
5. 创始人团队是否公开自己的共同责任与分歧。
6. Vale是否把网络交给中央、撤销中央签名、暂时误导招募线，或建立持续的人工治理。

### 明确的最终问题

Seedline不是一台可以彻底摧毁的中央机器。玩家最终决定的是下一版规则的权力来源：公司合规、临时欺骗、单方面撤销、公开分支，或持续人工看护。任何成功都必须说明谁得到保护、谁承担维护、哪些危险副本仍然存在。

## Continuity

### 技术硬规则

- Seedline只处理电话路由、版本、权限、日志与人工签名；不写成有意识的网络。
- 礼仪层不能改变天气、制造事件或瞬间控制未经校准的普通人。
- 工作记忆分区只限制同一身体中的记忆访问，不创造克隆、灵魂或平行人格。
- MCE-0交班磁带不保存人格或记忆本体；它保存重新打开同一大脑夜班记忆访问的声学检索钥匙、口述交班与正式登记内容。
- Vale的私人自我与工作自我都是真实的Evelyn，双方掌握不同生活范围；故事不把其中一方写成假人。
- 四秒人工窗口需要具体故障、时间和可验证现象；Leonard不能凭私人信任无限开启。
- Seedline存在联网、离线和主动拒绝同步的分支副本。中央线路断开只会停止认证更新，不会让所有地方副本同时消失。
- Wren运营签名决定中央兼容表与合同范围；Vale安全签名确认拒绝、投诉和人工入口。两项签名都在时，联网分支才自动更新。
- 最终发布或撤销必须使用Vale安全签名、外部托管凭据和至少一项独立事件证据；Leonard的技术签字提供可审计效力，但不替代创始人签名。

### 人物知识边界

- 第一章开场Vale不知道私人生活、姓名和项目来源，但保留拨号、工单与线路判断技能。
- Maeve只知道医院流程、转运事故和自己听见的话，不知道创始人历史。
- Peter知道夜班楼层、绩效和员工分区的实际感受，不知道全部源代码或外部托管。
- Leonard知道技术架构、旧故障和Vale身份，但不知道她私人自我留下的全部计划。
- Dorothy知道文件来源与认领条件，不懂线路内部实现。
- Wren知道项目历史、Vale分区安排和中央发布计划，但不能预知独立人物的每次选择。

### 叙事清晰度

- 每次通话先给出对方的当前工作、当前目标和直接风险。
- 每章最多正式解释一个新系统概念：礼仪层、员工分区、MCE-0交班、外部托管、双签名、版本治理。
- 重要术语出现后必须马上被人物用于行动，不能只停留在说明。
- 所有跨夜后果必须通过日期、人物记忆、工单状态或实体物件至少展示一项。
- 替代分支和七结局互斥，故事账本不得把它们当作同一时间线同时发生。
