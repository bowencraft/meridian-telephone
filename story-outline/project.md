# 电话亭：完整互动叙事

## Premise

1968年的一个伦敦雨夜，一名不记得自己姓名的人进入一座不在地图上的红色电话亭。玩家通过拨号、接听陌生来电、检查亭内物件和积累跨夜班线索，逐步发现公共天气、失物招领、午夜广播与礼仪培训都通向Meridian Courtesy Exchange。

主角最终发现自己是被Meridian拆散并删除姓名的十九号接线员E. Vale。Vale曾参与设计能够逐步改写语言习惯与身份记忆的“礼仪协议”，后来试图关闭它。故事的核心承诺，是让玩家先破解一张电话网络，再拼回一个人的身份，并决定如何处理自己曾帮助创造的系统。

## Core Question

当主角发现自己既是Meridian的受害者，也是它的早期参与者时，会选择恢复自己的身份、关闭网络，还是接管网络去保护仍在线上的人？

## Constraints

- Point of view: 第二人称玩家视角；通话对端允许有限的独立人物视角信息，但不直接切换叙述者。
- Tense: 现在时。
- Target length: 六章非线性互动正文；单次夜班约15–30分钟，完整长线真相通过连续多个雨夜逐步拼合。
- Intended audience: 喜欢叙事解谜、模拟交互、心理悬疑和多结局探索的玩家。

## Narrative Clarity

- 核心专有名词只保留Meridian、MCE-0电话亭、礼仪协议和十九号席位。
- 每一阶段只承担一个主要问题：发生了什么、号码为何相连、谁可信、主角是谁、最后如何选择。
- 大多数来电者都是拥有自己目标的独立人物；只有被明确标记为回声的少数线路属于E. Vale的记忆碎片。
- 独立人物必须能够帮助、阻拦、误导、拒绝或偶然路过，不能只负责播报线索。
- 抽象设定先通过人物行动、通话后果或实体物件展示，再用一句清楚的话解释。
- 重大揭示和结局只能使用此前出现过的人物选择、线索与网络规则。
- 所有“重新开始”都发生在更晚的夜班；人物与世界后果持续，只有临时线路权限重置。
