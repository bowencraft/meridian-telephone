# 电话亭：Seedline夜班

## Premise

1989年的伦敦雨夜，一名不记得私人生活的夜班拨号者在一座已从公共目录注销的红色电话亭里接线。玩家通过拨号、接听真实员工与市民来电、检查交班物件和积累跨夜证据，逐步发现医院、交通、投诉和呼叫中心都运行在同一套模块化线路工具Seedline上。

主角最终发现自己是Seedline共同设计者Evelyn Vale被隔离出来的工作自我。Vale曾为保护高压岗位员工而设计工作记忆分区，却允许Meridian把它与统一话术、绩效和公共服务记录绑定。故事的核心承诺，是让玩家先解决一场具体的雨夜转运，再追查一套理想主义技术如何变成控制工具，并决定应由谁继续维护一个已经无法彻底收回的网络。

## Core Question

当主角发现自己既是被隔离的员工，也是分区技术的创造者时，会把Seedline交还中央机构、撤销中央授权、公开安全版本，还是留下承担持续的人工作业？

## Constraints

- Point of view: 第二人称玩家视角；通话对端允许有限的独立人物视角信息，但不直接切换叙述者。
- Tense: 现在时。
- Target length: 六章非线性互动正文；单次夜班约20–35分钟，完整长线真相通过连续多个雨夜逐步拼合。
- Intended audience: 喜欢叙事解谜、模拟交互、心理悬疑和多结局探索的玩家。

## Narrative Clarity

- 核心专有名词只保留Meridian、Seedline、礼仪层、工作记忆分区和MCE-0。
- 每一阶段只承担一个主要问题：发生了什么、号码为何相连、谁可信、主角是谁、最后如何选择。
- 大多数来电者都是拥有自己目标的独立人物；只有被明确标记为回声的少数线路属于E. Vale的记忆碎片。
- 独立人物必须能够帮助、阻拦、误导、拒绝或偶然路过，不能只负责播报线索。
- 抽象设定先通过医院转运、员工绩效、故障工单或实体物件展示，再用一句清楚的话解释。
- 重大揭示和结局只能使用此前出现过的人物选择、线索与网络规则。
- 所有“重新开始”都发生在更晚的夜班；Vale的工作自我由MCE-0交班磁带延续，其他人自然记得前夜，只有当班权限重置。
